Regarding 'ascending order'

From a developer's point of view, 'ascending' depends on the context. Ascending value can change depending on what game is being played. Of course, a 9 will always be greater than a 1, but the value of aces, jacks, kings, and queens can be different across games.

For this example, I used the 'standard' order of cards: 2-10, jack, queen, king, and ace. The order of the suits is clubs, diamonds, hearts, and then spades.

As far as the code is concerned, this is just an array of numbers. I've written a function that converts them into the names of the cards. It contains an array named 'deckOrder' if the order of the cards ever needs to be changed.

Unit Tests

I wrote unit tests in Jasmine, and ran them through Karma. If you have Karma installed already, you can navigate to this folder in the command line and just run karma start.
